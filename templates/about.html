The Project

Happy California receives live, geotagged tweets filtered for the Bay Area from the Twitter Stream API. Some tweets from Southern California are included on the map due to Twitter users' ability to tag their tweet with a "place" in addition to the geotag. If the latitude longitude coordinate boundary box of the "place" they specify overlaps with the Bay Area, and the tweet is geotagged, it passes through the Twitter Stream API's filter.

The tweets are analyzed by a pickled NLTK Naive Bayes classifier for positive or negative sentiment as they are received from the Twitter Stream API. Pusher Websocket API listens for new tweets, and pushes the analyzed tweets to the frontend. Each tweet is also stored in Mongo DB.

The frontend receives the tweet, checks if the tweet is positive or negative, and maps the tweet with the appropriate emoticon using Leaflet/Mapbox. The user is then able to click on each mapped emoticon to view the text of the tweet. In addition, the user is able to click on the control buttons to view only positive tweets, only negative tweets, all tweets, or clear the entire map to start over.

When the page first loads, the frontend detects the current hour of the day and loads all tweets from 12 am today to the present hour. The user can then adjust the slider to view tweets by other hour ranges.

For the sentiment analysis, a training set of 1.6 million pre-labeled tweets was used. The set was obtained from Sentiment140 and can be found here. Each tweet was preprocessed to remove any usernames, links, and articles included in the text, then tokenized. The tokenization process I used preserved emoticons so that the presence and absence of positive or negative emotions could be used in the sentiment analysis.
